Order

— обслуживание клиента по созданному им заказу


Задачи

  • Беглая оценка ордера и помощь в принятии решений
  • Инструмент для продажи VAS
  • Менеджмент писателей выполняющих заказ
  • Хранилище информации (привязанной к заказу)
Экран ордера не предназначен для управления процессом написания заказа.
Из этого процесса ордер интересуется только тремя состояниями:
  • Процесс идет
  • Работа закончена
  • Что-то мешает процессу — ордер обязан сказать что мешает и дать инструмент для решения проблемы


Рождение ордера

На продуктовых сайтах есть ордер-форма. Клиент нажимает submit и создает order.

Содержание созданного ордера:
  • клиент — все, что система знает о нем
  • содержимое ордер-формы (заказ)
  • платеж для клиента
